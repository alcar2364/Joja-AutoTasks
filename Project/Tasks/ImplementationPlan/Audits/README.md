# Audits

This folder stores audit, triage, and parking-lot artifacts that support an
active or recently completed implementation phase.

## Contents

- In-progress audit reports and parking-lot artifacts that should not live in
  the ImplementationPlan root beside execution checklists.
- Review or triage artifacts that document findings without becoming canonical
  issue-tracker records on their own.

## Boundaries

- Keep atomic execution checklists in the parent `ImplementationPlan/` folder.
- Move canonical unresolved or archived implementation issues into
  `../ImplementationIssues/`.
- Do not recreate legacy deferment tracking artifacts in this folder.
